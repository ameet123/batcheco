Loco Batch Flow Steps

  • Step-0: Offer Criteria File Load/Parse

    Load/parse offer criteria file and create a domain model for it. Build corresponding SQL & store it in local configuration object as well as database

  • Step-1: Criteria verification

    Verify the SQL generated for syntax accuracy. Also check that the criteria for each file are recognizable

  • Step-2: Pre-processing file parser

    Load and parse the pre-processing file and generate a domain model. Save the model in configuration object and the database

  • Step-3: Pre-processing verification

    Verify the sequence of steps and the cchronology. Ensure the accuracy.

  • Step-4: Pre-proceeessing SQL generation

    Based on the domain model, generate SQL for each offer. This sql is responsible for creating the offer-specific file with the uniform desired format, i.e. the list of columns.

  • Step-5: Execute pre-processing SQL in Hadoop

    Generate the offer specific table data for each offer using Hive SQL

  • Step-6: Offer Criteria SQL execution

    execute the offer criteria and generate the final daily offer table based on the SQL generated in the very first step.

  • Step-7: Extract, consolidate and pack

    Extract the data from Hadoop to local directory, consolidate it into a single file and compress it if desired. This is now ready for shipping.
